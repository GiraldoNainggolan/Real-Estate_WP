Hai di sana. Nama saya Carla dan saya berada di tim pendidikan pelanggan di Asana. Dalam kursus ini, saya senang menunjukkan kepada Anda beberapa cara Anda dapat merampingkan pekerjaan tim Anda dan memaksimalkan produktivitas Anda menggunakan Asana untuk manajemen kerja. Pekerja di seluruh industri kehilangan waktu yang tak terhitung jumlahnya untuk bekerja tentang pekerjaan: waktu yang terbuang untuk mencari informasi, beralih antar aplikasi, dan mengadakan rapat status. Alat manajemen kerja seperti Asana dirancang untuk mengurangi waktu yang terbuang dengan memberdayakan manajer proyek untuk fokus pada perencanaan proyek, penyelarasan tim, dan alokasi sumber daya. Banyak organisasi menggunakan jenis alat ini untuk mengelola segala sesuatu mulai dari proyek sederhana dengan tim kecil, hingga proyek kompleks dengan banyak pemangku kepentingan. Hal yang paling saya sukai dari Asana adalah hal itu memberi saya ketenangan pikiran sepenuhnya bahwa tidak ada yang akan jatuh melalui celah-celah. Saya dapat melacak semua tugas saya sendiri di satu tempat dan rekan tim saya dan saya selalu jelas tentang siapa yang bertanggung jawab atas apa, dan kapan itu perlu dilakukan. Saya akan muncul di seluruh kursus ini untuk men demonstrasikan kegiatan manajemen proyek umum menggunakan Asana. Dalam video ini, saya akan menunjukkan cara membangun Backlog Produk yang sama untuk Virtual Verde yang baru saja Anda buat. Kali ini, Anda akan menggunakan proyek, tugas, sub tugas, dan bidang khusus di Asana. Saya akan menunjukkan contoh dari setiap langkah, sehingga Anda tahu persis bagaimana hal itu dilakukan. Dan dalam aktivitas berikut, kami akan memberi Anda gambaran rinci dan instruksi untuk setiap langkah, sehingga Anda dapat mencobanya sendiri di Asana. Siap? Mari kita mulai. Pertama-tama, saya akan log ke akun Asana saya. Selanjutnya, saya akan membuat proyek untuk Backlog Produk saya menggunakan template perencanaan sprint di Asana. Asana memiliki banyak template bermanfaat untuk semua jenis proyek, sehingga Anda dapat menjelajahi beberapa opsi lain sendiri. Selanjutnya, saya akan mengisi detail proyek. Dalam hal ini, saya akan memperbarui nama menjadi Virtual Verde dan membuat proyek. Saya tidak perlu memperbarui hal lain untuk aktivitas ini, tetapi jika Anda mau, Anda dapat memilih opsi untuk tim Anda. Setelah Anda membuat proyek, Anda akan diarahkan ke tampilan papan, di mana Anda dapat mulai menambahkan detail proyek. Tampilan papan dapat membantu Anda melacak rencana Sprint tim Anda mirip dengan menggunakan papan Kanban. Sebagian besar tim membuat proyek di awal setiap Sprint dan kemudian melacak pekerjaan mereka melalui setiap tahap, untuk memastikan mereka tahu di mana posisi pekerjaan Sprint mereka. Dalam template ini, tahapan tersebut diwakili oleh bagian yang muncul sebagai header kolom dalam tampilan papan. Anda akan melihat bahwa banyak templat Asana, seperti perencanaan Sprint ini, memiliki tugas yang sudah ditambahkan. Untuk keperluan demo ini saya akan menghapus ini tetapi Anda dapat meninjau sendiri jika Anda ingin mempelajari lebih lanjut tentang cara menggunakan template ini. Sekarang saya telah membuat proyek Backlog Produk saya, saya akan memasukkan judul cerita pengguna dengan menambahkan tugas ke kolom Backlog untuk proyek Virtual Verde. Ju@@ dul cerita pengguna pertama saya adalah “opsi perawatan rendah,” lalu “alat perawatan tanaman” dan saya akan melakukan satu lagi, “bantuan dan saran ahli.” Bagus, semuanya siap. Berikut daftar lengkap judul cerita pengguna saya. Selanjutnya, saya akan menambahkan cerita pengguna. Untuk melakukan itu, saya harus membuka panel rinci tugas dan memperbarui deskripsi tugas dengan cerita. Ingat membuat cerita pengguna di aktivitas terakhir? Saya akan menambahkan salah satunya sebagai contoh di sini di bawah “deskripsi.” Saya akan memilih kisah pengguna ini: Sebagai pelanggan potensial, saya ingin mencari tahu tanaman mana yang paling mudah dirawat sehingga saya dapat membeli opsi perawatan rendah. Selanjutnya, saya perlu menambahkan kriteria penerimaan untuk cerita pengguna. Karena Anda memasukkan judul cerita pengguna sebagai tugas, Anda akan menambahkan kriteria penerimaan sebagai sub tugas. Untuk kriteria penerimaan pertama, saya akan menambahkan “kemampuan untuk menyortir tanaman berdasarkan pemula, menengah dan lanjutan.” Untuk kriteria selanjutnya, “kemampuan untuk mencari tanaman dengan kebutuhan perawatan yang sama.” Ketika saya selesai menambahkan sub tugas, saya akan menutup panel terperinci sehingga saya dapat mengakses semua tugas di Backlog. Sebagai tip cepat, saya dapat memperluas daftar sub tugas dan melihat kriteria penerimaan saya tanpa membuka panel detail. Dan yang tak kalah pentingnya untuk aktivitas ini, saya akan menambahkan bidang khusus untuk judul epik. Bidang kustom memungkinkan Anda menambahkan detail tambahan ke tugas dalam proyek. Anda dapat menganggapnya seperti tag atau kolom dalam spreadsheet. Untuk kegiatan ini, saya akan memilih bidang drop-down dan menggunakan “epic” sebagai judul bidang dan menambahkan “inisiatif perawatan tanaman” sebagai opsi pertama. Saya hanya perlu satu opsi untuk aktivitas ini, jadi saya akan menghapus opsi dua. Dan selesaikan pembuatan lapangan. Karena semua cerita ini adalah bagian dari epik yang sama, saya dapat memperbarui epik untuk semua cerita pengguna sekaligus. Sekarang setelah saya menambahkan semua detail proyek ini, saya dapat memeriksa kartu tugas yang telah selesai yang menunjukkan kepada saya kriteria penerimaan cerita pengguna dan epik semuanya di satu tempat. Setelah saya selesai menambahkan kriteria penerimaan untuk semua cerita pengguna saya, papan Asana saya muncul seperti ini. Saya telah berhasil membuat Backlog Produk saya untuk proyek Virtual Verde.
