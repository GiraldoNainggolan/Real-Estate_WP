Hei di sana. Jika Anda melewatkan dua video terakhir saya di mana saya menunjukkan kepada Anda cara membuat Backlog dan menambahkan estimasi di Asana, lihat saja. Dalam video ini. Saya akan menunjukkan cara membuat dan mengelola sprint di Asana. Saya akan membawa Anda melalui contoh setiap langkah sehingga Anda tahu persis bagaimana hal itu dilakukan. Dengan merencanakan Sprint di Asana, tim dapat memiliki kejelasan penuh tentang rencana Sprint, tonggak sejarah, tanggal peluncuran, dan Backlog, dengan upaya kerja dan komunikasi bersama di satu tempat. Kemudian, dalam kegiatan berikut kami akan memberi Anda petunjuk terperinci sehingga Anda dapat mencobanya sendiri. Mari kita mulai. Saya akan mulai dengan membuka Virtual Verde Backlog yang saya buat di aktivitas sebelumnya, menambahkan estimasi di Asana. Dalam kegiatan ini kita akan memprioritaskan dan men jadwalkan setiap item di Backlog ke Sprint mendatang. Jadi saya akan menambahkan bidang khusus untuk Sprint, yang akan memungkinkan saya untuk dengan mudah mengetahui Sprint mana yang dijadwalkan setiap tugas. Saya akan menambahkan “Sprint” ke judul bidang dan mengonfirmasi jenis bidang sebagai drop-down. Kemudian saya akan mengubah opsi satu menjadi “Current Sprint” dan opsi dua menjadi “Next Sprint.” Sekarang saya akan menetapkan item ke Backlog Sprint untuk dua Sprint pertama. Untuk langkah ini, saya sarankan beralih ke tampilan daftar, karena lebih mudah untuk melihat semua tugas Anda sekaligus. Saya akan menetapkan item ke Sprint saat ini atau Sprint berikutnya, menggunakan drop-down di kolom Sprint. Dan kemudian saya akan mengurutkan ulang proyek berdasarkan Sprint, untuk mengelompokkan item Sprint saya bersama-sama. Secara default, Asana akan mengurutkan tugas saya dalam bagian yang telah saya buat. Jika saya menonaktifkan penyortiran di dalam bagian, saya akan melihat tugas saya dikelompokkan berdasarkan bidang khusus yang saya gunakan untuk mengurutkan. Dalam hal ini, dengan Sprint. Saya juga akan melihat jumlah bidang kustom numerik di bagian bawah setiap grup. Ini memudahkan untuk melacak berapa banyak poin yang telah saya berikan untuk setiap Sprint sehingga saya memastikan saya tidak melebihi kapasitas tim saya. Dan akhirnya, saya akan menambahkan tanggal jatuh tempo untuk setiap tugas. Masih banyak lagi yang dapat Anda lakukan dengan perencanaan Asana untuk Sprint. Anda dapat menyesuaikan detail proyek Anda, merencanakan Sprint Anda dengan timeline, mengelola beban kerja Sprint, dan memantau kemajuan proyek Sprint. Perencanaan Sprint hanyalah salah satu cara Asana dapat membantu Anda mengelola proyek dan tugas tim. Dari mencapai tujuan dan tenggat waktu hingga melacak kemajuan dan pemecahan masalah, mengotomatiskan dan merampingkan pekerjaan Anda, dan menyatukan semua aliran kerja Anda di satu tempat , Asana memiliki banyak sumber daya untuk memastikan semua proyek Anda siap untuk sukses. Sekarang giliran Anda, pergilah ke item kursus berikutnya di mana Anda akan menemukan instruksi terperinci untuk menyelesaikan aktivitas ini di Asana sendiri.
